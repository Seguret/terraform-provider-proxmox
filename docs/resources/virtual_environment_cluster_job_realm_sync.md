# proxmox_virtual_environment_cluster_job_realm_sync (Resource)

Manages a Proxmox VE cluster realm-sync scheduled job.

## Example Usage

### Sync Users Daily

```terraform
resource "proxmox_virtual_environment_cluster_job_realm_sync" "ldap_users" {
  realm    = "company-ldap"
  schedule = "daily"
  scope    = "users"
  enabled  = true
  comment  = "Daily LDAP user synchronization"
}
```

### Sync Groups at 3 AM

```terraform
resource "proxmox_virtual_environment_cluster_job_realm_sync" "ldap_groups" {
  realm    = "company-ldap"
  schedule = "3"  # 3 AM daily
  scope    = "groups"
  enabled  = true
  comment  = "Nightly LDAP group synchronization"
}
```

### Sync Both Users and Groups

```terraform
resource "proxmox_virtual_environment_cluster_job_realm_sync" "ldap_sync" {
  realm          = "company-ldap"
  schedule       = "0 2 * * *"  # 2 AM daily
  scope          = "both"
  enabled        = true
  enable_new     = true
  remove_vanished = "entry,properties"
  comment        = "Full LDAP synchronization with cleanup"
}
```

### AD Realm Sync

```terraform
resource "proxmox_virtual_environment_cluster_job_realm_sync" "ad_sync" {
  realm          = "company-ad"
  schedule       = "*/30"  # Every 30 minutes
  scope          = "users"
  enabled        = true
  enable_new     = false
  remove_vanished = "entry"
  comment        = "Continuous AD user updates"
}
```

## Schema

### Required

- `realm` (String) The authentication realm to synchronize.
- `schedule` (String) The job schedule in systemd calendar format (e.g. 'daily', '0 3 * * *').

### Optional

- `comment` (String) A description for this sync job.
- `enable_new` (Boolean) Whether to enable newly synced users.
- `enabled` (Boolean) Whether the sync job is enabled.
- `remove_vanished` (String) Comma-separated list of objects to remove when they vanish from the LDAP directory (acl, entry, properties).
- `scope` (String) What to sync: 'users', 'groups', or 'both'.

### Read-Only

- `id` (String) The realm-sync job identifier.
