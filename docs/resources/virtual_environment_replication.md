# proxmox_virtual_environment_replication (Resource)

Manages a Proxmox VE replication job. The job ID is formed as '{vmid}-{job_id}'.

## Example Usage

### Simple VM Replication

```terraform
resource "proxmox_virtual_environment_replication" "vm_100_replica" {
  source_vm_id = 100
  target_node  = "pve2"
  job_id       = 0
  enabled      = true
  comment      = "Replicate VM 100 to secondary node"
}
```

### Scheduled Replication with Bandwidth Limit

```terraform
resource "proxmox_virtual_environment_replication" "vm_with_schedule" {
  source_vm_id = 101
  target_node  = "pve3"
  job_id       = 0
  schedule     = "*/30"  # Every 30 minutes
  rate         = 50     # 50 MB/s limit
  enabled      = true
  comment      = "Continuous replication with throttling"
}
```

### Multiple Replications for Same VM

```terraform
resource "proxmox_virtual_environment_replication" "primary_replica" {
  source_vm_id = 102
  target_node  = "pve2"
  job_id       = 0
  schedule     = "*/15"
  enabled      = true
  comment      = "Primary replication target"
}

resource "proxmox_virtual_environment_replication" "secondary_replica" {
  source_vm_id = 102
  target_node  = "pve3"
  job_id       = 1
  schedule     = "0 */4"  # Every 4 hours
  rate         = 30      # Lower priority
  enabled      = true
  comment      = "Secondary replication for DR"
}
```

### Disabled Replication (Maintenance)

```terraform
resource "proxmox_virtual_environment_replication" "maintenance" {
  source_vm_id = 103
  target_node  = "pve2"
  job_id       = 0
  enabled      = false
  comment      = "Temporarily disabled for maintenance"
}
```

## Schema

### Required

- `job_id` (Number) The numeric job ID within the VM (e.g. 0, 1).
- `source_vm_id` (Number) The VMID to replicate.
- `target_node` (String) The destination node name.

### Optional

- `comment` (String) A comment for the replication job.
- `enabled` (Boolean) Whether the replication job is enabled.
- `rate` (Number) Bandwidth limit in MB/s.
- `schedule` (String) Cron schedule for the replication job (e.g. '*/15').

### Read-Only

- `id` (String) The replication job ID (format: vmid-jobid, e.g. '100-0').
