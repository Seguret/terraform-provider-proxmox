# proxmox_virtual_environment_firewall_options (Resource)

Manages firewall options for cluster, node, VM, or container.

## Example Usage

### Cluster Firewall Configuration

```terraform
resource "proxmox_virtual_environment_firewall_options" "cluster" {
  scope        = "cluster"
  enabled      = true
  policy_in    = "DROP"
  policy_out   = "ACCEPT"
  log_ratelimit = "enable=1,rate=1/second,burst=5"
}
```

### Node Firewall Options

```terraform
resource "proxmox_virtual_environment_firewall_options" "node" {
  scope      = "node/pve"
  enabled    = true
  policy_in  = "DROP"
  policy_out = "ACCEPT"
  dhcp       = true
  ndp        = true
}
```

### VM Firewall Configuration

```terraform
resource "proxmox_virtual_environment_firewall_options" "vm_firewall" {
  scope      = "vm/pve/100"
  enabled    = true
  policy_in  = "DROP"
  policy_out = "ACCEPT"
  dhcp       = true
  mac_filter = false
  ip_filter  = true
}
```

### Container with Advanced Firewall Options

```terraform
resource "proxmox_virtual_environment_firewall_options" "container" {
  scope         = "ct/pve/200"
  enabled       = true
  policy_in     = "REJECT"
  policy_out    = "ACCEPT"
  dhcp          = true
  ebtables      = true
  mac_filter    = true
  ip_filter     = true
  ndp           = true
  log_ratelimit = "enable=1,rate=5/second,burst=10"
}
```

## Schema

### Required

- `scope` (String) The firewall scope: 'cluster', 'node/<node>', 'vm/<node>/<vmid>', 'ct/<node>/<vmid>'.

### Optional

- `dhcp` (Boolean) Whether to allow DHCP traffic.
- `ebtables` (Boolean) Whether to enable Ethernet bridge filtering.
- `enabled` (Boolean) Whether the firewall is enabled.
- `ip_filter` (Boolean) Whether to enable IP filtering.
- `log_ratelimit` (String) Log rate limit (e.g., 'enable=1,rate=1/second,burst=5').
- `mac_filter` (Boolean) Whether to enable MAC address filtering.
- `ndp` (Boolean) Whether to enable NDP (Neighbor Discovery Protocol).
- `policy_in` (String) Default inbound policy (ACCEPT, DROP, REJECT).
- `policy_out` (String) Default outbound policy (ACCEPT, DROP, REJECT).

### Read-Only

- `id` (String) The ID of this resource.
