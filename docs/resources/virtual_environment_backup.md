# proxmox_virtual_environment_backup (Resource)

Manages a Proxmox VE vzdump backup schedule.

## Example Usage

### Daily Backup of All VMs

```terraform
resource "proxmox_virtual_environment_backup" "daily_all" {
  storage     = "local-lvm"
  schedule    = "02:00"  # 2 AM daily
  all         = true
  compress    = "zstd"
  enabled     = true
  comment     = "Daily backup of all VMs and containers"
}
```

### Selective VM Backup

```terraform
resource "proxmox_virtual_environment_backup" "production_vms" {
  storage     = "backup-storage"
  schedule    = "Mon,Wed,Fri 02:00"
  vmid        = "100,101,102"
  compress    = "zstd"
  enabled     = true
  comment     = "Production VMs - 3x per week"
  mailto      = "ops@example.com"
  mail_notification = "failure"
}
```

### Bandwidth-Limited Backup

```terraform
resource "proxmox_virtual_environment_backup" "throttled" {
  storage     = "backup-storage"
  schedule    = "daily"
  all         = true
  compress    = "zstd"
  bw_limit    = 10240  # 10 MB/s limit
  enabled     = true
  comment     = "Bandwidth-limited backup to avoid network saturation"
}
```

### Backup with Email Notifications

```terraform
resource "proxmox_virtual_environment_backup" "monitored" {
  storage               = "pbs-datastore"
  schedule              = "daily"
  all                   = true
  compress              = "zstd"
  enabled               = true
  mail_notification     = "always"
  mailto                = "backup-admin@example.com,ops-team@example.com"
  comment               = "Full cluster backup with notifications"
}
```

## Schema

### Required

- `schedule` (String) The backup schedule in systemd calendar format (e.g. 'daily', 'Mon,Tue 02:00').
- `storage` (String) The target storage for backups.

### Optional

- `all` (Boolean) Back up all VMs and containers.
- `bw_limit` (Number) Bandwidth limit in KiB/s.
- `comment` (String) A description for this backup job.
- `compress` (String) Compression algorithm ('0', '1', 'gzip', 'lzo', 'zstd').
- `enabled` (Boolean) Whether the backup job is enabled.
- `mail_notification` (String) Email notification mode ('always', 'failure').
- `mailto` (String) Comma-separated list of email addresses to notify.
- `max_files` (Number) Maximum number of backups to keep (deprecated; use prune settings).
- `mode` (String) Backup mode ('snapshot', 'suspend', 'stop').
- `nodes` (String) Comma-separated list of nodes to run the job on. Empty means all nodes.
- `notes_template` (String) Template for backup notes.
- `vmids` (String) Comma-separated list of VMID(s) to back up. Leave empty when 'all' is true.

### Read-Only

- `id` (String) The backup job ID (assigned by Proxmox).
