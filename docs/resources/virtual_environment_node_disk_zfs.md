# proxmox_virtual_environment_node_disk_zfs (Resource)

Manages a ZFS pool on a Proxmox VE node.

## Example Usage

### Simple ZFS Mirror Pool

```terraform
resource "proxmox_virtual_environment_node_disk_zfs" "mirror" {
  node_name   = "pve"
  name        = "tank"
  devices     = "/dev/sdb,/dev/sdc"
  raid_level  = "mirror"
  add_storage = true
}
```

### ZFS RAID-Z Pool

```terraform
resource "proxmox_virtual_environment_node_disk_zfs" "raidz" {
  node_name   = "pve"
  name        = "storage"
  devices     = "/dev/sdb,/dev/sdc,/dev/sdd"
  raid_level  = "raidz"
  compression = "lz4"
  add_storage = true
}
```

### ZFS RAID-Z2 with Compression

```terraform
resource "proxmox_virtual_environment_node_disk_zfs" "raidz2" {
  node_name   = "pve"
  name        = "raidz2-pool"
  devices     = "/dev/sdb,/dev/sdc,/dev/sdd,/dev/sde"
  raid_level  = "raidz2"
  compression = "zstd"
  ashift      = 12
  add_storage = true
}
```

### Single Device ZFS Pool

```terraform
resource "proxmox_virtual_environment_node_disk_zfs" "single" {
  node_name   = "pve"
  name        = "single-disk-pool"
  devices     = "/dev/nvme0n1"
  raid_level  = "single"
  compression = "zstd"
  add_storage = true
}
```

## Schema

### Required

- `devices` (String) Comma-separated list of device paths to use for the ZFS pool (e.g. /dev/sdb,/dev/sdc).
- `name` (String) The ZFS pool name.
- `node_name` (String) The node name.

### Optional

- `add_storage` (Boolean) Whether to automatically add the created ZFS pool as a Proxmox storage.
- `ashift` (Number) Pool sector size exponent (ashift value, e.g. 12 for 4K sectors).
- `compression` (String) ZFS compression algorithm (e.g. lz4, zstd, on, off).
- `raid_level` (String) The RAID level (mirror, raidz, raidz2, raidz3, single, draid, draid2, draid3).

- `id` (String) The ID of this resource.
