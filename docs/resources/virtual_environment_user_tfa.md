# proxmox_virtual_environment_user_tfa (Resource)

Manages a two-factor authentication (TFA) entry for a Proxmox VE user.

## Example Usage

### TOTP-Based 2FA

```terraform
resource "proxmox_virtual_environment_user_tfa" "admin_totp" {
  user_id     = "admin@pve"
  type        = "totp"
  description = "Admin TOTP authentication"
  enabled     = true
}
```

### Recovery Codes

```terraform
resource "proxmox_virtual_environment_user_tfa" "backup_codes" {
  user_id     = "admin@pve"
  type        = "recovery"
  description = "Backup recovery codes"
  enabled     = true
}
```

### WebAuthn (FIDO2)

```terraform
resource "proxmox_virtual_environment_user_tfa" "webauthn" {
  user_id     = "admin@pve"
  type        = "webauthn"
  description = "Security key authentication"
  enabled     = true
}
```

### Multiple 2FA Methods for User

```terraform
resource "proxmox_virtual_environment_user_tfa" "admin_totp" {
  user_id     = "admin@pve"
  type        = "totp"
  description = "Primary TOTP method"
  enabled     = true
}

resource "proxmox_virtual_environment_user_tfa" "admin_backup" {
  user_id     = "admin@pve"
  type        = "recovery"
  description = "Recovery codes backup"
  enabled     = true
}

resource "proxmox_virtual_environment_user_tfa" "admin_security_key" {
  user_id     = "admin@pve"
  type        = "webauthn"
  description = "YubiKey 5"
  enabled     = true
}
```

## Schema

### Required

- `type` (String) The TFA type. Valid values: 'totp', 'recovery', 'webauthn', 'yubico'.
- `user_id` (String) The Proxmox user ID (e.g. 'root@pam').

### Optional

- `description` (String) A human-readable description for the TFA entry.
- `enabled` (Boolean) Whether the TFA entry is enabled.
- `totp` (String, Sensitive) The TOTP provisioning URI or secret used during initial setup. Not returned after creation.
- `value` (String, Sensitive) The proof value (e.g. current TOTP code) used to validate the TFA entry during creation.

### Read-Only

- `id` (String) The TFA entry identifier assigned by Proxmox.
- `qrcode` (String, Sensitive) The QR code data URI returned on creation (TOTP/yubico types). Sensitive.
- `url` (String) The provisioning URL returned on creation for TOTP types.
